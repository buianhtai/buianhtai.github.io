# Frontmatter Schema for Blog Writer

Use this exact schema for posts generated by the `blog-writer` skill.

## Required Fields

### `title`
- Type: `string`
- Required: Yes
- Rule: Clear, specific, and architecture-focused.

### `description`
- Type: `string`
- Required: Yes
- Rule: One-sentence summary of what the post explains.

### `pubDate`
- Type: `string` (format `YYYY-MM-DD`)
- Required: Yes
- Rule: Must be ISO date only, no timestamp.

### `tags`
- Type: `string[]`
- Required: Yes
- Rule: 3-8 lowercase tags; include `architecture`.

### `lang`
- Type: enum
- Required: Yes
- Allowed values: `"en" | "vi"`

### `category`
- Type: enum
- Required: Yes
- Allowed value for this skill: `"architecture"`

## Optional Fields

### `series`
- Type: enum
- Required: No
- Allowed values: `"goclaw" | "ai-agent-tools"`
- Rule: Include only when post belongs to a sequence.

### `seriesOrder`
- Type: `number`
- Required: No
- Rule: Required when `series` is present; positive integer preferred.

### `heroImage`
- Type: `string`
- Required: No
- Rule: Path or URL to an image asset.

## Canonical Example

```yaml
---
title: "NVIDIA OpenShell: How Sandboxed Runtimes Make AI Agents Safe"
description: "A deep dive into OpenShell runtime architecture, policy enforcement, and privacy-aware inference routing."
pubDate: 2026-03-22
tags: ["ai", "rust", "architecture", "security", "agents"]
lang: "en"
category: "architecture"
series: "ai-agent-tools"
seriesOrder: 5
heroImage: "/images/blog/openshell-architecture.png"
---
```

## Validation Notes
- Keep date format strict (`YYYY-MM-DD`) to satisfy Astro date coercion.
- `series` and `seriesOrder` should appear together for ordered navigation.
- Do not add unsupported fields.
