Most agent demos look convincing until they need to survive a retry, pause for human approval, or resume with state that outlives a single request. LangGraph is the layer built for that harder problem.

<LangGraphHeroInfographic />

<Callout color="green">
  Prompts are inputs to nodes. State is the system of record. That is the architectural shift LangGraph makes explicit.
</Callout>

          <li>branching based on structured intermediate state,</li>
          <li>checkpointing before risky or expensive actions,</li>
          <li>human approval in the middle of the run,</li>

    <div class="prose prose-lg max-w-none">
      <p>
        The project from <a href="https://github.com/langchain-ai/langgraph">langchain-ai/langgraph</a> treats agents
        less like chatbots and more like workflow applications that happen to use LLMs inside some steps.
      </p>
      <p>
        The center of gravity shifts away from the prompt and toward the graph around it. Nodes become explicit
        application steps. Edges define control flow. Shared state becomes the contract that every step reads and
        updates.
      </p>
      <Callout color="amber">
        LangGraph is most useful when the application has durable state and explicit transitions. If the problem is just
        one model call plus one tool, it is probably too low-level.
      </Callout>
    </div>
  </div>
